Installation

1. Install packages

npm:

To install the library using npm, run the following command in your project directory:

npm install @blizz-ui/core @blizz-ui/components

This will download and install the required packages.

yarn:

If you prefer to use yarn, run the following command in your project directory:

yarn add @blizz-ui/core @blizz-ui/components

Yarn will fetch and install the necessary packages.

2. Import module

Regardless of whether you used npm or yarn for installation, the module import remains the same. In your Angular project, import the BlizzModule from the @blizz-ui/components package. Add the following import statement: 

import { BlizzModule } from '@blizz-ui/components';

Then, include the module in the imports array of your app's main module using the forRoot method: 

BlizzModule.forRoot()

3. Customization

If you want to customize the library, you can pass a config object as an argument to the forRoot method. To create a custom configuration, follow these steps:

  1. Create a new file in the 'src' folder of your Angular project, for example blizz.config.ts.
  2. Define a constant variable blizzConfig and assign it an object that implements the BlizzConfig interface. This object should contain the desired customization options. For example: 
    import { BlizzConfig } from '@blizz-ui/components';

export const blizzConfig: BlizzConfig = {
  base: 'blizz',
  theme: 'crystal',
  components: {}
};
  3. In your app's main module, import the custom configuration file: 
    import { blizzConfig } from './blizz.config';
  4. Pass the imported blizzConfig object as an argument to the forRoot method: 
    BlizzModule.forRoot(blizzConfig)

By following these steps, you can customize the library according to your requirements.

Remember to import and use the components from the library in your Angular application as needed. You can refer to the Components documentation for detailed information on the available components and their usage.

4. Config

The BlizzConfig interface is used to define the configuration options for customizing the Blizz UI library.

BlizzConfig
Property Type Default Description
base BlizzPredefinedConfig 'blizz' Represents the base configuration for the Blizz UI library.
Currently available predefined configs: 'blizz', 'material'.
theme BlizzPredefinedTheme | BlizzConfigTheme 'crystal' Represents the theme configuration for the Blizz UI library. It can be a predefined theme or a custom theme.
Currently available predefined themes: 'crystal'.
components BlizzConfigComponentsDictionary - Represents the configuration options for individual components in the library. It is a dictionary that maps component keys to their respective configuration objects.

The BlizzConfig interface allows you to customize various aspects of the Blizz UI library, including the base configuration, theme, and component-specific configurations. By providing a custom configuration object implementing the BlizzConfig interface, you can tailor the library's appearance and behavior to suit your application's needs.

Note: The configuration objects for individual components, such as BlizzConfigChip, BlizzConfigTextField, BlizzConfigIcon, etc., define component-specific configuration options. These options allow you to customize the styles, states, variations, and elements of each component.

Please refer to the Components documentationfor more detailed information on the available configuration options and their usage.

4.1. Creating Custom Theme

The BlizzConfigTheme type represents the theme configuration for the Blizz UI library.

The BlizzPredefinedTheme type defines the available predefined themes for the library. Currently, the only predefined theme is 'crystal'.

The BlizzConfigTheme type allows you to specify a predefined theme as the base theme for the library. Additionally, you can include any custom theme configurations that conform to the BlizzTheme type. Custom theme can be any nested object where final value is string. However, we recommend creating your custom themes with use of BlizzThemeConfig interface. Thanks to this, there won't be need of manually specifying colors for every component, to make sure color exists in the theme.

BlizzThemeConfig
Property Type Description
white string Represents the color code for the "white" color in the theme palette.
black string Represents the color code for the "black" color in the theme palette.
neutral BlizzColorShade Represents the shades of neutral colors in the theme palette.
primary BlizzColorShade Represents the shades of primary colors in the theme palette.
secondary BlizzColorShade Represents the shades of secondary colors in the theme palette.
tertiary BlizzColorShade (optional) Represents the shades of tertiary colors in the theme palette.
success BlizzColorShade Represents the shades of success colors in the theme palette.
warning BlizzColorShade Represents the shades of warning colors in the theme palette.
info BlizzColorShade Represents the shades of info colors in the theme palette.
error BlizzColorShade Represents the shades of error colors in the theme palette.
text string Represents the default color code for text in the theme.
textOpposite string (optional) Represents the opposite color code for text in the theme. This is an optional property.

The BlizzColorShade type represents the shades of a specific color in the Blizz UI theme palette. It is a dictionary that maps color shade keys (BlizzColorShadeKey) to their respective color codes (string). The BlizzColorShadeKey type is an enumeration of possible shade keys: 50 (usually lightest), 100, 200, 300, 400, 500, 600, 700, 800, 900, 950 (usually darkest). It provides a way to specify different shades of a color in the palette. Each key maps to a specific color code representing the respective shade of the color.
Additionally you can specify optional default property, 500 is default if not specified. It represents the default shade key for the color. If specified, this shade key will be used as the default shade for the color.

4.2. Using Theme Colors

You can use theme colors directly in style files via css variables, for example if you want to use color primary and it's 600 shade, you can use:

var(--bzz-theme_primary-600)

All the theme color variables are prefixed with --bzz-theme_, and the value is flatten object separated with dashes (-).

If you want to use the default shade, the shade key can be skipped, so for example if you want to use the default shade of primary color, you can use:

var(--bzz-theme_primary)

4.3. Customizing Components

The BlizzConfigComponentsDictionary type is a dictionary that holds the configuration options for individual components in the Blizz UI library. It maps component keys to their respective configuration objects.

Each key in the dictionary represents a specific component, while the associated value is the configuration object containing customization options and settings for that component. The configuration object for each component extends the BlizzConfigComponent interface.

The BlizzConfigComponent interface defines the structure for component-specific configuration options, including elements, states, and variations. It allows you to customize the styles, behavior, and other properties of each component.

BlizzConfigComponent
Property Description
elements Represents the configuration options for the elements within the component. Each element has styles property, that includes all properties possible to customize the style of the element.
states Represents the configuration options for the different states of the component's elements. Each state is dictionary of component elements. States may be native html states such as: hover and disabled, or component specific custom states. Specifying properties in a state overrides default state properties.
variations Represents the configuration options for the variations of the component's elements and states. Specifying properties in a variation overrides default variation properties.

To fully understand and utilize the available configuration options for each component, it is recommended to refer to the Components documentation, which provide detailed explanations and usage guidelines.

4.4. Customizer

The "Customizer" feature in the Blizz UI library is a powerful tool that enables users to generate component configurations easily using a user-friendly graphical user interface (GUI) similar to popular design tools like Figma.

With the "Customizer," users can visually modify various aspects of a component's configuration, such as elements styles, states, and variations, without needing to manually edit configuration file. The GUI provides an intuitive interface where users can interact with controls, sliders, color pickers, and other visual elements to customize the component's appearance and behavior.

The feature simplifies the process of generating component configurations by offering a more visual and interactive approach. Users can experiment with different settings, instantly preview the changes, and generate the corresponding configuration output based on their customizations. It helps users can rapidly iterate and fine-tune the appearance of components to match their desired design or branding. It enhances the flexibility and customization capabilities of the Blizz UI library, making it easier for developers and designers to create visually appealing and consistent user interfaces.

To access the "Customizer" feature, simply navigate to the documentation page of the desired component. There, you will find a "Customize" button. Clicking on the button will open the "Customizer" interface.
